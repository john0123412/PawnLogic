# PawnLogic 0.2.1 Post-Release Stabilization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stabilize the 0.2.0 consolidation release by expanding install, CLI, provider, stream, tool, workspace, metrics, docs, and release-guard regressions before shipping 0.2.1.

**Architecture:** Keep every public contract stable while turning 0.2.0's new internal boundaries into stronger regression surfaces. Prefer deterministic tests, narrow helpers, conservative typed-island expansion, and release checks that protect package metadata and documentation consistency.

**Tech Stack:** Python 3.10+, pytest with temporary `PAWNLOGIC_HOME`, Ruff, mypy typed island, GitHub Actions, setuptools dynamic versioning, `build`, `twine`, and PyPI Trusted Publishing through GitHub Actions OIDC.

---

## Release Theme

0.2.1 is a post-release stabilization patch. It should prove that the 0.2.0
turn-state, provider-stream, trust-boundary, runtime-metrics, skill-pack, and
release-guard changes are reliable across install and runtime entry points.

## Non-Goals

- Do not add new model providers or model aliases.
- Do not add public commands, CLI flags, or config file formats.
- Do not change provider/model visibility rules.
- Do not change the public provider stream delta dict schema.
- Do not change tool result message shape, assistant message shape, or
  `reasoning_content` persistence rules.
- Do not introduce remote telemetry or send runtime metrics over the network.
- Do not package third-party skill packs into wheels, sdists, Docker images, or
  generated bundled-skill directories.
- Do not replace `AgentSession.run_turn()` with a new runtime framework.
- Do not rename English docs or introduce `_EN` filenames.

## Current Baseline

- 0.2.0 is published to PyPI and documented in `CHANGELOG.md`.
- `main` is synchronized with `origin/main` at the post-release closure commit.
- `v0.2.0` exists and points to the 0.2.0 release-prep commit.
- `README.md`, `README_zh-CN.md`, `CHANGELOG.md`, and `SECURITY.md` all describe
  `0.2.0` as the current public release.
- The 0.2.0 implementation added internal turn state, provider stream adapters,
  trust-boundary helpers, runtime metrics, skill-pack metadata boundaries, and
  broader typed-island coverage.
