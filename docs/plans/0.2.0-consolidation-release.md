# PawnLogic 0.2.0 Consolidation Release Implementation Plan

> **For agentic workers:** Implement this plan task-by-task. Keep each task in a
> focused local commit, run the listed verification before committing, and do not
> push, tag, or publish unless the maintainer explicitly authorizes that step.

**Goal:** Ship one 0.2.0 consolidation release that absorbs the planned 0.1.8
maintenance hardening, 0.1.9 provider/stream convergence, and the first 0.2.0
runtime architecture milestone without changing public CLI, provider, stream,
tool-result, or persistence contracts.

**Architecture:** Deepen existing internal modules instead of adding new public
runtime surfaces. Keep `config/paths.py:VERSION` as the only version source of
truth, make release consistency checks derive expectations from that version,
continue extracting `AgentSession.run_turn()` bookkeeping into small internal
helpers and state snapshots, isolate provider stream parsing behind internal
adapters while preserving the public delta dict schema, centralize trust-boundary
decisions for tool execution, and record lightweight runtime metrics without
changing user-facing terminal output by default.

**Tech Stack:** Python 3.10+, pytest with temporary `PAWNLOGIC_HOME`, Ruff, mypy
typed island, GitHub Actions, setuptools dynamic versioning, `build`, `twine`,
and PyPI Trusted Publishing through GitHub Actions OIDC.

---

## Release Theme

0.2.0 is a consolidation release. It should make the release path harder to get
wrong, reduce the size and risk of the main turn loop, prepare provider stream
logic for future adapters, and define packaging boundaries for optional skill
packs while preserving all public runtime behavior.

## Non-Goals

- Do not change public CLI command syntax, option names, default startup mode,
  or command completion semantics.
- Do not change provider/model visibility rules. DeepSeek remains active by
  default, custom providers remain inactive by default, and `/model` only shows
  configured visible models.
- Do not change the public provider stream delta dict schema.
- Do not introduce a public stream event class, dataclass protocol, or new
  runtime dependency for streaming.
- Do not change tool result message shape, tool-call JSON shape, or assistant
  message shape.
- Do not change `reasoning_content` persistence, restore, or provider-stripping
  rules.
- Do not replace `AgentSession.run_turn()` with LangGraph or another runtime
  framework.
- Do not add new model providers, bundled tools, or public commands.
- Do not package third-party skill packs, third-party skill Markdown, or copied
  upstream skill support files into the PyPI wheel, generated source archives,
  Docker images, or generated bundled-skill directories.
- Do not rename default English docs or add `_EN` filenames.

## Current Baseline

- 0.1.7 is released to PyPI and documented in `CHANGELOG.md`.
- `README.md` / `README_zh-CN.md` and `GUIDE.md` / `GUIDE_zh-CN.md` use the
  formal `zh-CN` filename suffix and are checked for matching heading structure.
- `tools/check_release_consistency.py` exists in CI but still contains
  historical-version assumptions that should be removed.
- `AgentSession.run_turn()` already delegates empty-response recovery,
  assistant/tool-call append, plan guard, concurrency truncation, tool execution,
  autosave checkpointing, and anti-loop injection to helpers.
- `core/tool_result.py` owns per-turn tool result counters.
- `core/turn_api.py` consumes provider stream delta dicts and preserves the
  public stream shape.
- Typed-island mypy currently covers selected core runtime and release-tooling
  files.
