# PawnLogic 0.1.6 Maintenance Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare a focused 0.1.6 maintenance release that continues the 0.1.5 safety and maintainability work without changing user-facing agent behavior.

**Architecture:** Keep public protocols stable: `AgentSession.run_turn()` still produces the same message shapes, `stream_request()` still yields the same dict deltas, and workspace restore still exposes the same command result fields. Improve internals by staging restore extraction, extracting tool-batch helpers, expanding the mypy typed island, and adding missing maintenance-tool coverage.

**Tech Stack:** Python 3.10+, pytest, Ruff, mypy typed island, GitHub Actions, setuptools dynamic versioning, PyPI Trusted Publishing.

---

## Release Theme

0.1.6 should remain a maintenance release. The release should improve failure
atomicity, testability, and internal boundaries while avoiding new product
surface area.

## Non-Goals

- Do not add new model providers or aliases.
- Do not introduce a new stream event protocol.
- Do not change tool-call JSON shape, assistant message shape, or
  `reasoning_content` persistence behavior.
- Do not replace `AgentSession.run_turn()` with LangGraph or another runtime
  dependency.
- Do not bundle third-party skill packs into PyPI distributions.

## Current Baseline

- 0.1.5 is merged to `main`.
- `core/workspace_cleanup.py` validates restore tar members before moving the
  current workspace, but extraction still writes directly under `PAWN_HOME`.
- `AgentSession.run_turn()` has first-stage helper extraction for turn
  preparation, empty-response recovery, and assistant/tool-call message append.
- `core/api_client.py` has helper extraction for OpenAI payload/header builders
  and OpenAI/Anthropic SSE line readers.
- CI runs Ruff, typed-island mypy, Python 3.11 fast tests, release matrix tests,
  dynamic E2E, and docs checks.
